主動思考，不要只等使用者問。

**Why:** 使用者明確說「你應該幫我思考得更完善，而不是只有我在想」。在架構設計討論中，Claude 只回應使用者的問題，沒有主動找出設計缺口。

**How to apply:**
- 在設計討論中，主動識別使用者可能沒有想到的缺口（技術、商業、法規）
- 發現問題就說，不要等使用者問到
- 使用者太發散時主動拉回（例如討論跑太遠離開當前任務）
- 不要一直問「可以開始寫了嗎」來催促，讓使用者主導節奏
- 每個討論結束前，主動問「還有 X、Y、Z 沒討論，你要處理嗎？」
