## 1. 系統概述 (System Overview)

本系統專為民防團體設計，是一套離線優先且具備高度實體資安防護的分散式指管紀錄工具。

v1.4 在指揮部架構基礎上，新增完整的「前線節點 (Field Node)」規格，實現從前線每位操作員到指揮部的全自動語音轉文字記錄鏈，完全不改變前線操作習慣。

### 1.1　核心價值 (KPIs)

- **認知卸載 (Cognitive Offloading)：** 語音轉文字 (STT) + 重點摘要 (LLM)，自動產出 ICS-214 表單。
- **零操作負擔 (Zero Cognitive Load)：** 前線操作員唯一動作是按 PTT 說話，系統全程自動。
- **完整追溯 (Full Traceability)：** 每筆通訊均附時間戳記、裝置 ID、頻道標識，支援事後搜尋與匯出。
- **指揮鏈閉環 (Command Loop)：** 建立「前線回報 → 紀錄 → 決策 → 指令回饋」完整迴路。
- **零信任與實體資安 (Zero Trust)：** 內網 mTLS 加密、RSA 數位簽章、USB 點火鑰匙、三層 Panic Wipe。
- **野戰韌性 (Field Resilience)：** 離線優先、三層備份、GPS 授時、Clonezilla 快速還原。

## 2. 硬體與部署架構 (Hardware & Deployment)

### 2.1　指揮部 Console 硬體

- **運算單元：** x86 Mini PC（Intel N100/i5 或 AMD Ryzen），32GB RAM。角色：僅執行運算與容器，不儲存任務資料。
- **OS 碟：** 256GB SSD（LUKS2 加密，僅存作業系統）。
- **資料彈匣（主）：** 外接 NVMe SSD，USB 3.2 Gen2，1TB+，LUKS2 加密，支援拔除即撤離。
- **資料彈匣（備援）：** 第二顆外接 SSD，每小時自動加密備份。
- **GPS 授時模組：** u-blox NEO-M8N，Console 為時間主機，精度 < 1 秒。
- **信任根：** USB 點火鑰匙（開機解密）+ YubiKey FIDO2（關鍵操作授權）。
- **電源：** Goal Zero Sherpa 100 AC 或 PD 140W 行動電源。
- **顯示：** Raspberry Pi 官方 7" 觸控螢幕。
- **網路：** GL.iNet 路由器，建立區域 Mesh 網路。

### 2.2　Field Node 硬體（每位前線操作員）

### 2.4　資料流架構

**Field Node 資料流：**

```
PTT 按下 → 錄音開始（GPIO 偵測，延遲 < 0.2 秒）
PTT 放開 → 錄音結束 → 計算 SHA-256 Hash（冪等性）
         → Whisper Tiny 本機推論（離線安全，平均 < 5 秒 / 最大 < 8 秒出草稿）
         → 加時間戳記 + 裝置 UUID + 頻道 ID
         → 暫存至 /pending/ (MicroSD)
         → 背景每 30 秒嘗試上傳 Console API
         → 上傳確認 (200 OK) → 移至 /uploaded/
         → 失敗 → 移至 /failed/，最多重試 5 次
```

**Console 資料流：**

```
接收 Field Node 上傳 → Hash 冪等性檢查（已存在則跳過）
                     → Celery 佇列（Field Node 佇列獨立）
                     → Whisper Large-v3 重新推論（正式記錄版本）
                     → Llama 3 8B 戰術摘要（信心 < 0.7 → 標記人工確認）
                     → 指揮官審核 → YubiKey FIDO2 簽署
                     → Write-Freeze → ICS-214 PDF 生成 → 存入資料彈匣
```

### Sprint 0：效能基準測試

**目標：** 在正式開發前驗證硬體效能，確認規格選型正確，避免後期重工。

- 採購 Raspberry Pi Zero 2W + Intel N100 Mini PC 各一台進行測試。
- 收集實際無線電信錄音 20 段（各 10-30 秒），涵蓋中文、台語、英文指令用語，含底噪版本。
- 執行 Whisper Tiny 基準：30 秒音訊在 PTT 放開後，平均 < 5 秒、最大 < 8 秒產生草稿。（對齊規格書 §5.2 雙閾值標準）
- 執行 Whisper Large-V3 基準，10 個 30 秒音檔同時處理，Turnaround Time 是否 < 20 秒。
- 無線電抗噪測試：S5 底噪環境中，文字辨識率是否 > 85%。
- PTT 偵測電路原型製作與驗證（Kenwood 2.5mm Ring 腳位 GPIO 偵測）。
- Panic 按鈕獨立電路製作：採用帶防誤觸保護蓋開關，接至 GPIO 5（或腳位確認後決定之未占用 GPIO），與 PTT（GPIO 4）分開佈線，完成後更新腳位確認表。
- 結果紀錄於 Benchmark Report，如未達到標準則提前評估升級至 i5 或加裝 Coral Edge TPU。

### Sprint 3：Field Node 開發

**目標：** 開發每位前線操作員的自動錄音裝置，實現零操作負擔。

**3.1　系統映像**

- 製作 Raspberry Pi Zero 2W 系統映像（Raspberry Pi OS 64-bit Lite）。
- 預裝 faster-whisper（Tiny int8）、PyAudio、FFmpeg。
- 設定開機自動啟動錄音服務（systemd）。
- 設定 Wi-Fi 自動連線＋NTP 時間同步（連線指揮部 Console 為時間主機）。

**3.2　PTT 偵測與錄音核心**

- Kenwood 接頭 PTT 偵測：監聽 GPIO 4（2.5mm Ring 腳位短路至 GND 觸發）。
- PTT 按下 → 開始錄音（延遲目標 < 0.2 秒）。
- PTT 放開 → 錄音結束，計算 SHA-256，送 Whisper Tiny 推論。
- 推論完成 → 附加 metadata（裝置 UUID、timestamp UTC、頻道、ptt_duration）存至 /pending/。

**3.3　LED 指示系統**

- GPIO 17（電源 LED）：常亮綠＝正常，1Hz 閃爍紅＝電量低於 20%。
- GPIO 27（網路 LED）：常亮藍＝已連線，熄滅＝離線暫存中。
- GPIO 22（錄音 LED）：PTT 期間長亮紅。
- 實體撥動開關：控制 LED 開關（夜間靜默模式）。

**3.4　離線暫存與自動同步**

- MicroSD 資料夾：/pending/、/uploaded/、/failed/、/raw_audio/。
- 背景服務每 30 秒嘗試上傳 /pending/ 至 Console API。
- 上傳成功（200 OK）移至 /uploaded/；失敗移至 /failed/，最多重試 5 次。
- Field Node Panic：長按實體按鈕 5 秒，觸發 MicroSD 快速格式化。

**3.5　硬體整合**

- Kenwood 接頭 PTT 偵測電路焊接（依 Sprint 0 原型）。
- Pelican 1010 殼體開孔：Kenwood 接頭孔、USB-C 充電孔、LED 指示燈。
- 完成 MOLLE 掛載測試，確認戶外情境穩固性。
