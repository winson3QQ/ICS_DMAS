## 三、發現的問題

### Bug #1：事件結案按鈕使用 native `confirm()` 對話框

- **嚴重度**：低（功能正確但 UX 不一致）
- **描述**：`closeIncidentWithConfirm(id)` 使用原生 `confirm()` 對話框，與 PWA 其他功能（如接收建檔）使用自訂模態框不一致。
- **影響**：
  1. 在瀏覽器自動化測試中無法操作 native dialog
  2. UX 風格不統一
- **建議**：改用與「確認建檔」相同的自訂模態框。

### Bug #2：JavaScript 按鈕點擊會誤觸同名按鈕

- **嚴重度**：低（測試方法造成，非正常操作路徑）
- **描述**：在治療區用 `startsWith('黑區')` 篩選按鈕時，因為底部導航也可能有包含「黑」的元素，導致 M003-MJ 意外從黃區移到黑區。
- **影響**：正常使用者不會遇到（需精確比對文字）。
- **建議**：（無需修改，此為自動化測試方法問題）

### Bug #3：物資管理「補給」按鈕使用 native `prompt()` 對話框

- **嚴重度**：低（功能正確但 UX 不一致）
- **描述**：點擊物資的「補給」按鈕會觸發 native `prompt()` 對話框要求輸入補給數量，導致頁面凍結（自動化測試無法處理）。
- **影響**：與其他功能使用自訂模態框的風格不一致。
- **建議**：改用自訂的數字輸入模態框。

### Bug #4：物資管理「確認盤點完成」疑似需要 native dialog 確認

- **嚴重度**：低
- **描述**：點擊「確認盤點完成」後，畫面進入物資使用模式（有取用/補給按鈕），但仍顯示「⚙ 尚未完成盤點確認」。推測確認流程依賴 native `confirm()`。
- **影響**：盤點確認狀態可能未正確設定。
- **建議**：確認是否使用了 native dialog，若是則改為自訂模態框。

### Bug #5：更多頁面底部導航按鈕點擊不響應

- **嚴重度**：中
- **描述**：從儀表板透過底部導航按鈕點擊「更多」（⋯），UI 上無法成功導航。需使用 JavaScript `switchTab('more')` 才能進入。
- **影響**：一般使用者可能無法透過觸控/點擊進入更多頁面。
- **建議**：檢查 `更多` 按鈕的 click 事件綁定是否正常，以及按鈕的 hit area 是否足夠大。

## 五、總結

醫療組 PWA v0.5.0-alpha 核心功能完整且穩定，START 檢傷分色、治療評估（XABCDE / SAMPLE / OPQRST）、後送 ISBAR 交接、物資管理、交班快照、QR 快照、加密匯出等功能均正常運作。

主要問題集中在 **native dialog 使用不一致**（Bug #1, #3, #4），建議統一改用自訂模態框，以提升 UX 一致性並避免在某些環境下的相容性問題。

**底部「更多」導航按鈕點擊不響應**（Bug #5）是唯一中度問題，建議優先修復。

極限測試顯示系統在 29 名傷患、紅區超載 167% 的情況下仍能正常運作，量能警示機制完善。
