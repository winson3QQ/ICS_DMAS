## 7. 結論與建議

### 判定

- [ ] **全部通過** → 採用 Gemma 4 E2B，進入 Phase 4 語音輸入整合
- [ ] **STT 通過，結構化不過** → Fallback：只用 E2B 做 STT，結構化用規則引擎
- [x] **結構化通過但延遲和矛盾偵測不過** → 見下方分析
- [ ] **全不通過** → 維持手動輸入，不整合語音 AI
- [x] **記憶體不足** → 7.2GB 模型在 8GB Pi 上餘裕極低

### 觀察到的問題

1. **延遲是硬體瓶頸**：BCM2712 + 8GB RAM 跑 7.2GB 模型，memory bandwidth 是推論速度的天花板，軟體無法最佳化
2. **矛盾偵測能力為零**：不是 prompt 調校能解決的程度，模型在指令遵從（填欄位）和批判性推理（抓矛盾）之間無法兼顧
3. **繁簡混用**：模型偶爾以簡體回應繁體輸入（如過敏欄位），影響下游比對

### Pi 500 跑 AI 的結論

**Pi 500 能跑 AI，但不能跑 7B+ 等級的模型做即時推論。**

- **結構化能力已驗證可行**（93.5%），但需要更小的模型（1B~4B）才能在 Pi 500 上達到 < 15 秒延遲
- **矛盾偵測應改用規則引擎**：GCS vs 意識、SpO2 vs 檢傷分類等邏輯是確定性的，規則引擎零延遲且 100% 可靠
- **STT 待測**：Whisper Tiny (~75MB) 在 Pi 500 上應可輕鬆運行，為下一步評估項目

### 後續行動

- [ ] 測試更小模型（Gemma3 1B/4B 或同級）在 Pi 500 上的結構化準確率 vs 延遲
- [ ] 安裝 faster-whisper + Whisper Tiny，用 stt_cases.json 測 CER
- [ ] 實作矛盾偵測規則引擎（純邏輯，不依賴 LLM）
- [ ] 改善評分腳本：chief_complaint 改用語意相似度而非嚴格字串比對
