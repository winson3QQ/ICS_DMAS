## 4. 元件規範

### 4.1 按鈕

**基礎**：`padding: 10px 18px; font-size: 15px; border-radius: var(--r-sm); font-weight: 600`

| 變體 | 樣式 | 使用場景 |
|------|------|---------|
| `btn-primary` | 金底白字 | 主要動作（確認、儲存） |
| `btn-outline` | 透明底金框金字 | 次要動作（列印、編輯） |
| `btn-danger` | 紅底白字 | 危險動作（離站、刪除、建事件） |
| `btn-success` | 綠底白字 | 成功動作（結案） |
| `btn-ghost` | 透明底灰字 | 取消、關閉 |
| `btn-sm` | 較小 padding/字級 | 內嵌小按鈕 |

### 4.2 Modal 底部按鈕規則

1. **所有按鈕使用 `flex-1`**（等寬）
2. **順序**：關閉(ghost) → 次要(outline) → 主要(primary) → 危險(danger)
3. **關閉在最左**，危險動作在最右
4. 單一按鈕情況：`btn-ghost flex-1`（如已離站/已結案的關閉按鈕）
5. **Auto-Save**：有輸入框的 modal，點任何操作按鈕時自動儲存輸入框文字，不需要獨立的「追加/儲存」按鈕

### 4.3 卡片

| 類型 | padding | border-radius | 用途 |
|------|---------|---------------|------|
| `.card` | 14px | `--r` | 設定區塊 |

- padding: `10px 12px`

## 5. 行為規範

### 5.1 灰階（停用/歷史）

所有「過去式」元素統一使用 `opacity: 0.5`：
- 已離站人員卡片
- 已結案事件卡片
- 暫停床位
- 關聯事件中的已結案項

### 5.2 紀錄格式

所有時間+操作員的紀錄統一格式：

```
時間(灰色) 操作員(金色粗體) — 內容
```

適用於：
- 事件處置紀錄
- 入站流程紀錄
- 人員備註
- 稽核日誌

顯示時：
- 時間：`color: var(--text2)`
- 操作員：`color: var(--primary); font-weight: 600`
- 內容：預設色

### 5.3 識別碼

格式：`S001-MA`（Shelter + 序號 + 性別代碼 + 年齡代碼）

| 性別代碼 | 意義 |
|---------|------|
| M | 男 |
| F | 女 |

| 年齡代碼 | 區段 |
|---------|------|
| I | 0-4 |
| J | 5-19 |
| Y | 20-34 |
| A | 35-49 |
| M | 50-64 |
| E | 65+ |

### 5.4 事件嚴重度預設

以下事件類型建立時預設「緊急」：
- `medical_infection`（醫療/傳染病）
- `air_raid`（空襲或飛彈攻擊）
- `security_threat`（攻擊或治安威脅）
- `medical_transfer`（需醫療轉介/轉送）

其他預設「普通」。

### 5.5 連動規則

| 觸發 | 連動 |
|------|------|
| 健康標記→已隔離 | 自動建立 `medical_infection` 緊急事件 |
| 健康標記→需關注 | 提示操作員選擇事件類型 |
| 健康標記→正常 | 追加備註至所有關聯未結事件 |
| 事件結案 | 彈出確認，詢問是否恢復涉及人員健康標記 |
| 離站 | 自動結案所有關聯未結事件 |
| 入站清消 | 自動產生識別碼 |
